# ZebraPose_hyx

## 介绍
- ZebraPose这个项目是instance level的识别方法，要制作一个目标检测网络的整体流程如下：
    1. 制作目标物体的3D彩色模型，这个一般是用激光扫描仪来做，比较贵的大概几万一台吧，也可以看看淘宝有没有代扫描的。做算法测试也可以直接用别人数据集里面的模型，然后去买对应的物体来做实验即可。
    2. 使用blenderproc制作BOP格式的数据集，这个是有制作好的pipeline的，编辑一下相机内参等参数一键生成几万张即可。
    3. 训练目标检测模型，例如YOLO或者FCOS等均可，这个比较成熟了。
    4. 训练位姿检测模型，预测目标物体可视范围内的每一个像素点的本地三维坐标编码。
    整体流程还是有点复杂，适合2-3人协作，或者一个人慢慢做。读懂paper和代码，虽然较为繁琐，但并不困难。
- ZebraPose的源代码较为凌乱，因此当时做了重构，核心逻辑没有变。
- 目前这个版本支持多GPU单网络单目标的训练，单网络支持多目标的代码由于时间太久找不到了 :)
- 不过如果理解了原理和代码结构，修改支持多目标并不复杂，主要有两个地方要修改：
    1. 单目标位姿检测网络的输入是前置目标检测网络的bbox扩展一定范围并缩放的截图，大小例如为torch.size([3,256,256])，需要让位姿网络知道该目标的id，因此最简单的方法是将输入改为torch.size([4,256,256])即可，多出来的torch.size([1,256,256])，其中的每一个数可以就是该目标id的embedding，这个embedding可以是学习出来的，也可以是定死的（0-1之间）。实验的结果是直接定死就可以。需要修改的文件是：
        a. 网络输入的大小：./networks/resnet.py，这里注意加载预训练resnet时也需要处理。
        b. dataloader：./datasets/bop_dataset_pytorch.py -> bop_dataset_single_obj_pytorch()
    2. 第二个地方是一个较为隐蔽的问题，这个问题由于没有做过实验，所以不能确定是否真的存在，但是当时也做了相应的处理，就是位姿检测网络对输入的截图会预测一个mask和对应的位置编码，在密集检测的场景下，如果截图中有两个相同id的物品，并且位置非常靠近，例如一个在前一个在后，而现在需要预测后面那一个的位姿，如果不对模型做提示，这时模型会非常疑惑到底要预测哪一个。因此这里需要对目标物体打标记，例如画一个bbox将物体框出来提醒网络，或者当时尝试的一个可行的办法是在bbox的xy中心点处做一个随机形状的标记等等。同理在训练截图上所做的修改也要应用到推理流程中。

    至此如果训练能够稳定收敛，在推理时逐一遍历目标检测网络检测到的物体即可。训练中实际上没有遇到过不稳定的情况。
- 另外还需要的bop_toolkit，git上下载即可。
- ZebraPose这个算法的想法非常精彩，我第一次看到时倒吸一口凉气。除此以外，loss的设计上还考虑到了对于那些确实难以预测的精度位置，如16bit中的后几位，如何不让模型去强行预测，这一点的做法也很有意思。

## 训练
- 编辑config.py, 主要是dataset path, data folder, object name等。
- 启动训练：
```python
torchrun --standalone --nproc_per_node=gpu/2/4 train_single.py/train_double.py
```
    gpu/2/4: 指使用所有可用GPU或2/4个GPU。
    train_single.py: 仅使用一个数据集做训练，对于网上的公开数据集，一般会叫train_pbr，指渲染出来的数据集。
    train_double.py: 使用两个数据集做训练，对于网上的公开数据集，一般是train_real和train_pbr，分别指真实拍摄和标注的数据集和渲染出来的数据集。
- 对于新的数据集，需要编辑tools_for_BOP/common_dataset_info.py
- 建议可以先用网上的公开数据集做训练，确定性能等满足预期后再自己制作数据集。

## 推理
- 推理部分的参考代码见inference，逻辑较为简单，只注意训练的时候对数据做了何种预处理，这里需要完全复刻。
- 对于预测精度，一个不可靠的仅供参考的数据是：1米内预测精度1%(仅translation)。
- 从经验来看，目标检测模型和位姿检测模型的速度都较快，速度瓶颈主要在最后的pnp处理，如果追求高精度，那么每一个mask内的像素本地3D位置预测结果都要送入pnp，如果减少这个预测点的数量，6D位姿预测速度可以快速提升，但预测精度下降，并且帧与帧之间会有明显抖动。
